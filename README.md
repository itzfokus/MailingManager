# Mailing Bot

Telegram-бот для массовой рассылки сообщений через подключённые боты. Управляется через интерфейс Telegram, поддерживает текст, фото, inline-кнопки и локальную базу получателей.

## Возможности

- Подключение нескольких ботов по токену
- Рассылка текстовых сообщений и фотографий с подписью
- Inline-кнопки двух типов: URL и Copy (скопировать текст)
- Предпросмотр сообщения перед запуском
- Загрузка списка получателей из `.txt`-файла или из локального `telegram.txt`
- Живой прогресс рассылки с кнопкой остановки
- Обработка FloodWait (429) с автоматическими повторами
- Доступ только для указанных владельцев (`OWNER_ID`)

## Установка

```bash
npm install
```

## Настройка

Создай файл `.env` в корне проекта:

```env
BOT_TOKEN=123456789:AAF...токен_управляющего_бота
OWNER_ID=123456789
```

Если владельцев несколько — перечисли через запятую:

```env
OWNER_ID=123456789,987654321
```

## Запуск

```bash
npm start
```

## Файл получателей

`telegram.txt` в корне проекта — список Telegram ID пользователей, по одному на строку:

```
8379662483
3333
2222
1111
```

Используется кнопкой **«Взять из локального хранилища»** на шаге загрузки файла. Альтернативно можно отправить любой `.txt`-файл прямо в чат с ботом.

## Команды

| Команда | Описание |
|---------|----------|
| `/start` | Главное меню |
| `/panel` | Список подключённых ботов |
| `/connect` | Подключить нового бота |

## Формат inline-кнопок

При добавлении кнопок к рассылке отправь текст в формате:

```
URL - https://t.me/username - Перейти
COPY - promo2024 - Скопировать промокод
```

Каждая кнопка — на отдельной строке. Формат: `ТИП - значение - подпись`.

## Структура проекта

```
MailingManager/
├── org/
│   ├── main.ts        # Логика бота
│   ├── db.ts          # SQLite (better-sqlite3)
│   ├── logger.ts      # Цветной консольный логгер
│   └── tsconfig.json
├── data.db            # База данных (создаётся автоматически)
├── telegram.txt       # Локальный список получателей
└── package.json
```
